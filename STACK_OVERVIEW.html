Visão Geral da Stack

Panorama consolidado das tecnologias, decisões arquiteturais e próximos passos do projeto Eyevital.

1. Stack Principal

Aplicação

  • SPA React 19
  • TypeScript
  • Vite (HMR / build rápido)

Estilos

  • Tailwind CSS v4
  • CSS legado modularizado
  • Fonte Inter (Google Fonts)

Qualidade

  • ESLint base
  • Type Checking TS
  • Error Boundary custom

Runtime

  • DOM API mínima (landing)
  • IntersectionObserver
  • Console logging estratégico

3. Componentes & Responsabilidades

PaginaInicial

  • Hero + seções informativas
  • Menu com estado visual baseado em rolagem (IntersectionObserver)
  • Gateway para autenticação via estado interno & query param ?auth=

AuthPage

  • Alternância Login / Registro (tabs acessíveis)
  • Validação progressiva de senha e confirmação
  • Estados derivados (passwordValid, mismatch gating)
  • Recursos de acessibilidade (aria-* e feedback tardio)

4. Validação do Formulário de Autenticação

  • Mínimo de senha: 6 caracteres
  • Mismatch: mostrado apenas após progresso suficiente ou blur
  • Can Submit: depende do modo (login vs registro) e ausência de erros
  • Estados: touched, confirmBlurred, submitting
Derivações puras UX progressiva Sem repetição de lógica

5. Acessibilidade

  • Uso de role="tablist" / role="tab" para o switch Login/Registro
  • aria-invalid e aria-describedby para feedback de erro
  • Botões toggle de senha com aria-pressed e rótulos contextuais
  • Estrutura semântica clara (headings, formulários)

6. Error Handling & Observabilidade

  • Error Boundary custom em main.tsx
  • Logs: [main], [App], [PaginaInicial], [AuthPage]
  • Borda de debug e heading de diagnóstico temporário

7. Decisões Arquiteturais

  • SPA única (remoção de múltiplos entrypoints)
  • Derivação de estado em vez de múltiplas flags redundantes
  • Manter legado marcado antes de excluir definitivamente
  • Query param simples para modo auth antes de adotar roteador
